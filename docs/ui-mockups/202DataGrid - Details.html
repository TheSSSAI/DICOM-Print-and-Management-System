Description

A comprehensive data grid organism for displaying, sorting, and filtering lists of complex data, such as DICOM studies. It includes states for loading, empty results, and errors, and is built for desktop applications.

Design System Alignment

The DataGrid is a composite organism that aggregates molecular components like SearchBar, Buttons, and list items. It heavily relies on the design system's typography, color palette for states, and spacing for its internal layout and structure.

Responsive Specifications

Mobile Adaptations

  • At <600px, header items stack vertically.
  • Less critical columns are hidden to avoid horizontal scrolling.

Tablet Adaptations

  • At <840px, secondary columns are hidden.

Desktop Enhancements

  • Full column visibility.
  • Hover effects for selectable rows.

Accessibility Features

Keyboard Support

The table container is scrollable. Rows are focusable if they contain interactive elements. Sortable headers are part of the tab order.

Screen Reader Support

Uses semantic `

`, ``, ``, `
`, and `` elements. `` elements use `scope='col'`. Loading states use `aria-busy='true'`.

Focus Management

Visible focus indicators are applied to interactive elements within the grid, and focus is contained within rows when navigating with arrow keys (if implemented).

Aria Attributes

  • aria-label for the table to provide context.
  • aria-busy='true' during loading.
  • aria-sort on headers to indicate sort direction.

Color Contrast

All text and UI states meet WCAG 2.1 AA contrast requirements.
