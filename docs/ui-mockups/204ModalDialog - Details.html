Description

A modal dialog is a window that appears on top of the main content, disabling all other interaction until the user addresses the dialog. It's used for critical information, user decisions, or focused tasks.

Design System Alignment

The component uses the design system's color palette for background and actions, typography for content hierarchy, spacing for internal layout, and shadow tokens for elevation.

Responsive Specifications

Mobile Adaptations

  • N/A (Desktop application). The width is set to 90% of viewport width to prevent horizontal overflow on smaller window sizes.
  • max-height is set to 90vh to ensure the dialog fits vertically and the body becomes scrollable if needed.

Tablet Adaptations

  • N/A

Desktop Enhancements

  • Size variants (small, medium, large) control the max-width for optimal content display.
  • Subtle scaling animation on open/close for a smoother user experience.

Accessibility Features

Keyboard Support

The `Esc` key should close the dialog. Tab/Shift+Tab should be trapped within the dialog. All interactive elements (close button, inputs, action buttons) are focusable.

Screen Reader Support

Uses `role='dialog'`, `aria-modal='true'`. The title is linked via `aria-labelledby` and body content via `aria-describedby`.

Focus Management

When opened, focus should be moved to the first interactive element inside the dialog. When closed, focus should return to the element that triggered it. (Requires JS).

Aria Attributes

  • role='dialog'
  • aria-modal='true'
  • aria-labelledby='modal-title-id'
  • aria-describedby='modal-body-id'

Color Contrast

All text and button combinations meet WCAG AA contrast requirements.
