Description

A specialized organism that renders a preview of the final print output. It displays DICOM images within a selected page layout, including headers, footers, custom logos, and de-identification overlays. The canvas scales proportionally to fit the available viewport.

Design System Alignment

This component uses the design system's color palette for the background and UI elements, spacing for layout, and typography for textual overlays. It also incorporates the skeleton loading pattern for its rendering state.

Responsive Specifications

Mobile Adaptations

  • N/A. Component is for a desktop application.

Tablet Adaptations

  • N/A. Component is for a desktop application.

Desktop Enhancements

  • The canvas and its content scale down proportionally to fit the available container size, maintaining the correct aspect ratio of the selected paper.
  • Uses max-width and max-height to ensure it never overflows its container while scaling.

Accessibility Features

Keyboard Support

While the canvas itself is primarily visual, any interactive elements within it (e.g., if re-ordering images) must be made keyboard-operable.

Screen Reader Support

The main canvas container should have an `aria-label` summarizing the current preview (e.g., 'Print preview: 2x2 grid on A4 Portrait paper'). Textual overlays like patient data are inherently accessible.

Focus Management

Focus should be managed logically if interactive elements are introduced.

Aria Attributes

  • aria-label for the main canvas container.
  • aria-busy='true' on the container when in the 'rendering' state to inform assistive technologies that content is loading.

Color Contrast

All textual overlays (headers, footers) must meet WCAG AA contrast ratio of 4.5:1 against the white paper background.
