Description

A primary vertical navigation component that provides access to the main sections of the application. It supports both an expanded state (icon and text) and a collapsed, icon-only state to maximize content area.

Design System Alignment

This component is built using the design system's color palette for backgrounds and active states, typography for labels, and the spacing system for padding and item heights. It fully incorporates the specified interaction patterns for focus and active states.

Responsive Specifications

Mobile Adaptations

  • N/A. Component is for a desktop application.

Tablet Adaptations

  • N/A. Component is for a desktop application.

Desktop Enhancements

  • The component's primary adaptation is between its expanded and collapsed states, which can be triggered by user action or in response to a narrow window width (e.g., <840px breakpoint).

Accessibility Features

Keyboard Support

The list of navigation links is fully navigable using the Tab key. Links are activated with Enter/Space. Arrow keys can be used for navigation if the parent list has `role='navigation'` and appropriate event handling.

Screen Reader Support

The component uses a semantic `

` element. Active links are marked with `aria-current='page'` to inform screen readers of the user's current location. Icons are decorative (`aria-hidden='true'`) as they are accompanied by text labels.

Focus Management

A clear, high-contrast focus ring appears on keyboard focus for all interactive elements (links, toggle button), respecting the design system's focus indicator pattern.

Aria Attributes

  • aria-current='page'
  • aria-label for the
    element
  • aria-label and aria-expanded for the collapse/expand toggle button

Color Contrast

All text and icon colors meet WCAG AA contrast ratios against their respective backgrounds in all states (default, hover, active).
