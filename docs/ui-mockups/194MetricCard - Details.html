Description

A card component used on dashboards to display a single, key metric with a title, a large value, and an icon. It includes states for loading and variants for indicating warning or critical thresholds.

Design System Alignment

Uses the design system's color palette for semantic states (warning, error), typography scale for titles and values, and spacing/shadow tokens for card layout.

Responsive Specifications

Mobile Adaptations

  • Cards will stack vertically on very narrow screens (< 600px).
  • The component is designed to be fluid within a grid container.

Tablet Adaptations

  • Cards will typically arrange in a 2 or 3 column layout depending on available width.

Desktop Enhancements

  • Hover effects provide visual feedback.
  • Cards can be arranged in wider multi-column layouts (e.g., 4 or 5 columns).

Accessibility Features

Keyboard Support

Not applicable as the card is non-interactive. If wrapped in a link/button, it inherits keyboard support.

Screen Reader Support

Using `role='region'` and `aria-labelledby` provides context to screen reader users, defining the card as a distinct, labeled section.

Focus Management

No focus management needed for a static display component.

Aria Attributes

  • role='region'
  • aria-labelledby

Color Contrast

Text colors (`--color-neutral-gray-900` and `--color-neutral-gray-500`) on white background meet WCAG AA contrast requirements. Semantic colors are used for non-text indicators (border/icon).
