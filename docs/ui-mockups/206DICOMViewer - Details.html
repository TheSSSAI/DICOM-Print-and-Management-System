Description

A complex organism for displaying and interacting with DICOM image series. It includes a main viewport for image rendering, a toolbar for manipulation tools, and a thumbnail strip for navigating between images in a series. It supports various states and layouts like side-by-side comparison.

Design System Alignment

This component integrates the design system's foundation for its interactive elements. The toolbar uses standard button styles, colors define states (active, hover), and spacing tokens ensure a consistent layout of its constituent parts. The overall container uses neutral colors to focus attention on the medical image.

Responsive Specifications

Mobile Adaptations

  • N/A. This is a desktop application component. The component is fluid and will resize within its parent container.

Tablet Adaptations

  • N/A

Desktop Enhancements

  • Full interactive experience with hover states on toolbar and thumbnails.
  • Keyboard accessibility for all interactive elements.
  • Layout adapts to available space within the application window.

Accessibility Features

Keyboard Support

Toolbar tools and thumbnail items are focusable and activatable via Tab and Enter/Space.

Screen Reader Support

Toolbar buttons should have accessible names (aria-label or similar). The viewport canvas itself requires an Automation Peer in WPF to describe its content and state (e.g., 'Image 1 of 12, zoomed to 150%'). Metadata overlays are readable text.

Focus Management

Visible focus indicators are applied to all interactive elements.

Aria Attributes

aria-current='true' or aria-selected='true' should be used on the selected thumbnail. The active tool button should have aria-pressed='true'.

Color Contrast

Metadata text on the viewport has a text shadow to ensure readability against varied image content. UI controls meet WCAG AA standards.
