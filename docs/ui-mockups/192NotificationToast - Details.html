Description

Provides non-blocking, temporary feedback to users about the status of an asynchronous operation. Toasts appear in a corner of the screen, deliver a concise message, and then disappear automatically or via user interaction. They are crucial for confirming actions like saving, printing, or exporting without interrupting the user's workflow.

Design System Alignment

Utilizes the design system's semantic color palette for variants (info, success, warning, error), typography scale for the message text, spacing system for padding, and elevation tokens for the card-like appearance.

Responsive Specifications

Mobile Adaptations

  • Component has a max-width to prevent it from spanning the entire viewport on larger screens.
  • The container is pinned to the corner of the viewport, ensuring it does not interfere with main content.

Tablet Adaptations

  • Behavior is consistent with desktop and mobile.

Desktop Enhancements

  • Smooth entry and exit animations enhance the user experience without being distracting.

Accessibility Features

Keyboard Support

The close button is focusable and can be activated with Enter/Space keys.

Screen Reader Support

Each toast has `role='status'` (or `role='alert'` for errors) and `aria-live='polite'` (or `aria-live='assertive'` for errors) so its content is automatically announced by screen readers. The close button has an `aria-label` for clarity.

Focus Management

Toasts are non-modal and do not trap focus, allowing users to continue their work. The close button has a clear focus indicator.

Aria Attributes

  • role='status'
  • role='alert'
  • aria-live='polite'
  • aria-live='assertive'
  • aria-label='Close notification'

Color Contrast

Text color is selected to meet WCAG AA requirements against each semantic background color. Dark text is used on the warning (yellow) background to ensure readability.
